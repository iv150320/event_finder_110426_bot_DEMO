# Event Finder v8.0.0

Реальный поиск мероприятий через HTTP-запросы (без web_fetch/web_search).

## Что нового в v3.5

- **Реальный поиск** — больше не генерирует план, а делает HTTP-запросы
- **KudaGo public API** — основной источник, бесплатный, без API-ключа
- **Парсинг вузов** — ВШЭ, РЭУ, МГУ (факультеты) через requests
- **Кэширование** — 30 минут TTL, автоматическое
- **Дедупликация** — по normalize(title) + date
- **Фильтрация по датам** — только события в заданном диапазоне
- **Telegram-формат** — чистый текст с эмодзи 📅🗓📊

## Использование

```bash
# По умолчанию: конференции в Москве на 30 дней
python3 event_search.py

# С параметрами
python3 event_search.py -t концерты -c спб -s 15.04.2026 -e 30.04.2026

# JSON-вывод (для обработки агентом)
python3 event_search.py --json -t выставки
```

### Параметры

| Параметр | Описание | По умолчанию |
|---|---|---|
| `-t`, `--topic` | Тематика | конференции |
| `-c`, `--city` | Город | Москва |
| `-s`, `--start-date` | Начальная дата (DD.MM.YYYY) | сегодня |
| `-e`, `--end-date` | Конечная дата (DD.MM.YYYY) | +30 дней |
| `-u`, `--include-universities` | Включить вузы | false |
| `-j`, `--json` | JSON-вывод | false |

## Источники

### KudaGo (основной)
- **API:** `https://kudago.com/public-api/v1.4/events/`
- **Города:** msk, spb, ekb, nsk, nnv, kzn, smr, krd, sochi, ufa, krasnoyarsk
- **Без API-ключа**, generous rate limit

## Технические детали

### Кэширование
- Файл: `cache.db` (SQLite) в директории скилла
- TTL: 30 минут
- Ключи: MD5 от параметров запроса
- Потокобезопасный доступ через threading.Lock

### Дедупликация
- Нормализация: lowercase, удаление спецсимволов, схлопывание пробелов
- Ключ: `normalize(title) + date`

### Обработка ошибок
- При недоступности источника — пропускаем, не крашимся
- Если ничего не найдено — выводим сообщение об этом
- Timeout: 15 секунд на запрос
